# WashRoute DB Health Check — 2026-06-18

## 🔴 RED — active degradation (DB still up, but queries are failing)

The database is **still responsive** — admin login and trivial queries work fine, so this is **not** a repeat of the May 26 full lockout. But real app queries are **timing out continuously right now** (roughly 2 per minute in the logs), and several core queries are averaging several seconds. This is the same pattern that preceded the May 26 outage, caught earlier this time.

**What to do now:** For immediate relief, bump the compute add-on (or restart) in the Supabase dashboard → Settings → Compute and Disk. The durable fix is the database index + RLS optimization work — the slow queries below need indexes so they stop burning IO.
